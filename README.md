# Zeroth Principles

> **Тип репо:** Foundation (единственный в своём роде — стоит над типологией)
> **Source-of-truth:** Да (для нулевых принципов и карты иерархии принципов)

---

## Что такое нулевые принципы

Нулевые принципы — самые общие транс-дисциплинарные ограничения, которые отсекают неправильные модели и делают рассуждения проверяемыми. Они работают **в любой области**: в инженерии, в науке, в бизнесе, в творчестве.

Нулевые принципы **дисциплинируют** все остальные уровни: первые, вторые, третьи. Кто их освоил — тот не путает корреляцию с причинностью, не строит моделей, которые рассыпаются при проверке, и не принимает решений на основе наитий.

> Источник: Левенчук А.И. «Нулевые, первые, вторые и третьи принципы» (2026-02-17)

---

## Нулевые принципы (6 штук)

### ZP.1. Аксиоматичность

Теории строятся по строгим правилам (аксиомы, доказательства), а не по наитию.

**Следствие:** Любое утверждение должно быть выводимо из явных предпосылок. «Мне так кажется» — не аргумент. «Вот аксиомы, вот вывод» — аргумент.

### ZP.2. Структурность и симметрии

Описывай через инварианты и структуры, а не через отдельные объекты.

**Следствие:** Ищи то, что НЕ меняется при трансформациях (инвариант). Если ты описал 10 объектов, но не описал их структуру и связи — ты ещё ничего не описал.

### ZP.3. Многомасштабность

Интегрируй мелкое, усредняй, ищи универсальное описание на разных масштабах.

**Следствие:** Одно и то же явление выглядит по-разному на разных масштабах (клетка vs орган vs организм). Хорошая модель работает на нескольких масштабах одновременно.

### ZP.4. Оптимизация

Ставь задачу как поиск экстремума функционала (вариационные принципы).

**Следствие:** Вместо «перебери все варианты» — сформулируй, что именно оптимизируешь, и ищи экстремум. Природа работает через оптимизацию (принцип наименьшего действия), инженерия — тоже.

### ZP.5. Вероятность и информация

Сложные системы описывай через вероятности, энтропию и информацию.

**Следствие:** Не «X произойдёт», а «X произойдёт с вероятностью P при условиях C». Неопределённость — не враг, а объект описания.

### ZP.6. Вычислительные ограничения

Учитывай, что можно реально вычислить и сделать при конечных ресурсах.

**Следствие:** Красивая модель, которую невозможно посчитать за разумное время — бесполезна. Ресурсы (время, память, деньги, внимание) всегда конечны.

## Иерархия принципов (карта)

```
Level 0: Нулевые принципы (Zeroth Principles)
│   Транс-дисциплинарные мета-ограничения
│   Фреймворк: нет (слишком фундаментальны)
│   Репо: ZP (это репо)
│
├── дисциплинируют ↓
│
Level 1: Первые принципы (First Principles)
│   Фундаментальные основания дисциплин
│   Фреймворк: FPF (принципы + фреймворк в одном бандле)
│   Репо: FPF
│
├── ограничивают ↓
│
Level 2: Вторые принципы (Second Principles)
│   Принципы предметных областей
│   Фреймворк: SPF (только форма и процесс, без самих принципов)
│   Принципы: Pack (каждый Pack = набор вторых принципов конкретного домена)
│   Репо: SPF + PACK-*
│
├── определяют ↓
│
Level 3: Третьи принципы (Third Principles)
│   Ситуационные, проектные принципы
│   Фреймворки: S2R и другие (несколько возможных фреймворков)
│   Принципы: Downstream (конкретные проекты, боты, курсы)
│   Репо: DS-* + FMT-s2r
```

### Что есть на каждом уровне

| Level | Принципы (содержание) | Фреймворк (форма) | Почему так |
|-------|----------------------|-------------------|-----------|
| 0 | Нулевые (ZP.1–ZP.6) | Нет | Слишком фундаментальны — 6 принципов, не нуждаются в формализации |
| 1 | FPF (холоны, BC, trust calculus, Γ-алгебра) | FPF (калькули, правила вывода) | Тяжёлые — без машинерии бесполезны, поэтому бандл |
| 2 | Pack (домен: DP, MIM, PD, ECO...) | SPF (11-шаговый процесс, шаблоны) | Много доменов — нужен единый стандарт формализации |
| 3 | Downstream (конкретные системы) | S2R, и другие | Много типов проектов — нужны разные форматы |

### Синонимы

| Термин в экосистеме | Уровень принципов | Пример |
|---------------------|-------------------|--------|
| **Pack** | = вторые принципы | PACK-digital-platform, PACK-MIM |
| **Downstream** | = третьи принципы | DS-aist-bot, DS-strategist-agent |
| **FPF** | = первые принципы + фреймворк | FPF (бандл) |
| **SPF** | = фреймворк для вторых принципов | SPF (только форма) |
| **S2R** | = один из фреймворков для третьих принципов | FMT-s2r |

### Fallback Chain (обновлённая)

```
Downstream (3-й) → Pack (2-й) → SPF (форма 2-го) → FPF (1-й) → ZP (0-й)
```

## Связь с FPF

FPF содержит первые принципы и опирается на нулевые. Связь:

| ZP | Проявление в FPF |
|----|--------------------|
| ZP.1 Аксиоматичность | Part A: Kernel Architecture (аксиоматическая структура холонов) |
| ZP.2 Структурность | Holon = structural unit, Bounded Contexts |
| ZP.3 Многомасштабность | Holarchy (вложенные холоны на разных масштабах) |
| ZP.4 Оптимизация | Part B: NQD-CAL (Novelty-Quality-Diversity optimization) |
| ZP.5 Вероятность | Trust Calculus (F + G + R), TGA |
| ZP.6 Вычислительные ограничения | Part C: Domain-specific calculi (конечные вычислимые модели) |

## Как пользоваться этим репо

1. **Новый человек в экосистеме** — начни с этого файла. Здесь карта всех уровней принципов.
2. **Проверка решения** — пройди по 6 нулевым принципам: не нарушаю ли? Построено ли на аксиомах (ZP.1)? Описана ли структура (ZP.2)? Учтены ли масштабы (ZP.3)? Есть ли оптимизация (ZP.4)? Учтена ли неопределённость (ZP.5)? Реально ли вычислить (ZP.6)?
3. **Спор о подходе** — вернись к нулевым принципам: какой из ZP.1–ZP.6 нарушается?
