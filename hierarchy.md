# Иерархия принципов

> Карта всех уровней принципов в экосистеме.

---

## Карта

```
Level 0: Нулевые принципы (Zeroth Principles)
│   Транс-дисциплинарные мета-ограничения
│   Фреймворк: нет (слишком фундаментальны)
│   Репо: ZP (это репо)
│
├── дисциплинируют ↓
│
Level 1: Первые принципы (First Principles)
│   Фундаментальные основания дисциплин
│   Фреймворк: FPF (принципы + фреймворк в одном бандле)
│   Репо: FPF
│
├── ограничивают ↓
│
Level 2: Вторые принципы (Second Principles)
│   Принципы предметных областей
│   Фреймворк: SPF (только форма и процесс, без самих принципов)
│   Принципы: Pack (каждый Pack = набор вторых принципов конкретного домена)
│   Репо: SPF + PACK-*
│
├── определяют ↓
│
Level 3: Третьи принципы (Third Principles)
│   Ситуационные, проектные принципы
│   Фреймворки: S2R и другие (несколько возможных фреймворков)
│   Принципы: DS (конкретные проекты, боты, курсы)
│   Репо: DS-*
```

## Что есть на каждом уровне

| Level | Принципы (содержание) | Фреймворк (форма) | Почему так |
|-------|----------------------|-------------------|-----------|
| 0 | Нулевые (ZP.1–ZP.6) | Нет | Слишком фундаментальны — 6 принципов, не нуждаются в формализации |
| 1 | FPF (холоны, BC, trust calculus, Γ-алгебра) | FPF (калькули, правила вывода) | Тяжёлые — без машинерии бесполезны, поэтому бандл |
| 2 | Pack (домен: DP, MIM, PD, ECO...) | SPF (11-шаговый процесс, шаблоны) | Много доменов — нужен единый стандарт формализации |
| 3 | DS (конкретные системы) | S2R, и другие | Много типов проектов — нужны разные форматы |

## Синонимы

| Термин в экосистеме | Уровень принципов | Пример |
|---------------------|-------------------|--------|
| **Pack** | = вторые принципы | PACK-digital-platform, PACK-MIM |
| **DS** | = третьи принципы | DS-aist-bot, DS-strategist-agent |
| **FPF** | = первые принципы + фреймворк | FPF (бандл) |
| **SPF** | = фреймворк для вторых принципов | SPF (только форма) |
| **S2R** | = один из фреймворков для третьих принципов | FMT-s2r |

## Fallback Chain

```
DS (3-й) → Pack (2-й) → Base.Принципы (SPF → FPF → ZP)
```

## Навыки в иерархии

Навыки (skills) — автоматизированные процедуры исполнения — находятся на уровне **DS (Level 3)**. Они порождаются принципами, а не заменяют их.

```
ZP    → как ДУМАТЬ         (инфраструктура мышления, перенос между доменами)
FPF   → как ОПИСЫВАТЬ      (корректность, строгие различения)
Pack  → ЧТО знать в домене (source-of-truth, доменный язык)
DS    → ЧТО ДЕЛАТЬ         (навыки, программы, инструменты)
```

**Почему навыков недостаточно:**
1. Навык процедурен — отвечает «как сделать ЭТО», но не «КАКУЮ процедуру выбрать»
2. Навыки не переносятся — навык хирурга бесполезен в менеджменте
3. «Ведро навыков» ≠ интегрированная способность (FPF B.2.4)
4. Навыки не порождают новых навыков — принципы порождают
